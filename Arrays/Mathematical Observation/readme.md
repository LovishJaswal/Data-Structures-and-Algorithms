# Mathematical Observation / Counting

## Core Idea

Don't simulate everything.

Look for a mathematical property, hidden observation, or counting
technique that lets you compute the answer directly.

---

## Recognition Triggers

Ask yourself:

- Can I count instead of generate?
- Is there a formula?
- Does the value range help?
- Can frequencies replace sorting?
- Do negative numbers change the answer?
- Is there a hidden invariant?
- Am I doing unnecessary work?

---

## Common Observations

### Formula

Use a known mathematical relationship.

**Example**
- Missing Number

---

### Sign Analysis

The largest numbers aren't always the best choice.

**Example**
- Maximum Product of Three Numbers

---

### Frequency Counting

Use counts instead of sorting when the value range is small.

**Example**
- Array Partition

---

## Problems Solved

| Problem | Key Observation |
|---------|------------------|
| Missing Number | Expected Sum − Actual Sum |
| Maximum Product of Three Numbers | Two negatives can produce the maximum product |
| Array Partition | Frequency counting replaces sorting |

---

## Optimization Mindset

1. Get a correct solution.
2. Remove unnecessary memory.
3. Find the bottleneck.
4. Use constraints to optimize.

---
