# Simulation

## Core Idea

Follow the process exactly as described.

Instead of searching for a shortcut, execute each operation in order
while maintaining the required state.

---

## Recognition Triggers

Ask yourself:

- Does the problem describe a sequence of operations?
- Am I simulating a real-world process?
- Does each step depend on the previous one?
- Is implementation more important than optimization?

---

## Common Observations

### Step-by-Step Execution

Execute every operation in the given order.

---

### State Management

Maintain only the information needed after each step.

Examples:

- Position
- Direction
- Score
- Boundaries
- Current State

---

### Boundary Handling

Most simulation problems fail because of incorrect handling of
indices, directions, or edge cases.

---

## Optimization Mindset

1. Understand the process before coding.
2. Simulate one operation at a time.
3. Keep the state updated after every step.
4. Prioritize correctness before optimization.

---

## Mental Checklist

Before coding, ask:

☐ What state do I need to maintain?

☐ What changes after each operation?

☐ Does the order of operations matter?

☐ Have I handled all boundary cases?

☐ Can I simulate the process directly?
