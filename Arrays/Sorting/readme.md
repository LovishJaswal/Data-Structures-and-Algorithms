# Sorting

## Core Idea

Rearrange the elements so that hidden relationships become obvious.

Sorting is usually a preprocessing step that simplifies searching,
grouping, merging, or comparison.

---

## Recognition Triggers

Ask yourself:

- Will ordering the elements simplify the problem?
- Do equal or similar values need to be grouped together?
- Can adjacent elements reveal the answer?
- Can sorting eliminate unnecessary cases?

---

## Common Observations

### Group Similar Elements

Equal or related values become adjacent after sorting.

---

### Adjacent Comparison

Many problems only require comparing neighboring elements after
sorting.

---

### Simplify the Logic

Sorting often converts a complex problem into a single linear
traversal.

---

### Custom Ordering

Sometimes the default ascending order isn't enough.
A custom comparator may reveal the desired order.

---

## Optimization Mindset

1. Solve the problem without sorting if possible.
2. Use sorting when it significantly simplifies the logic.
3. After sorting, aim to solve the remaining problem in one pass.
4. Ask whether sorting is the actual bottleneck.

---

## Mental Checklist

Before coding, ask:

☐ Will sorting simplify the problem?

☐ Do adjacent elements reveal the answer?

☐ Can I finish in one traversal after sorting?

☐ Do I need a custom comparator?

☐ Can sorting be replaced by a better technique?
